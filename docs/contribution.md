# Contribution & Project History Files

These files help contributors understand how to participate in your project and track its evolution over time.

## CONTRIBUTING

Guidelines for contributing to the project. Essential for projects accepting external contributions.

### Required Contents
- Process for submitting issues and feature requests
- Pull request process and requirements
- Code style guidelines and conventions
- Testing requirements and procedures
- Development environment setup instructions
- Branch naming conventions
- Commit message guidelines

### Naming Variants
- `CONTRIBUTING.md`
- `CONTRIBUTING`
- `HACKING.md` (common in GNU/FSF projects)
- `HACKING`
- `DEVELOPMENT.md`
- `DEVELOPMENT`

### When to Include
- Projects accepting contributions
- Open source projects
- Team projects with multiple developers
- Optional for personal/experimental projects

## CHANGELOG

Record of all notable changes to the project. Critical for users and maintainers to track project evolution.

### Required Contents
- Version numbers with release dates
- Changes organized by category (Added, Changed, Deprecated, Removed, Fixed, Security)
- Notable bug fixes
- Breaking changes highlighted
- Deprecation warnings

### Format Standards
- Use Keep a Changelog format
- Organize chronologically (newest first)
- Include version numbers following semantic versioning
- Date format: ISO 8601 (YYYY-MM-DD)

### Naming Variants
- `CHANGELOG.md`
- `CHANGELOG`
- `HISTORY.md`
- `HISTORY`
- `NEWS.md`
- `NEWS`
- `RELEASES.md`
- `RELEASES`
- `CHANGES.md`
- `CHANGES`

## AUTHORS / CONTRIBUTORS / MAINTAINERS

List of people who contributed to the project.

### Required Contents
- Names of contributors
- Optional: email addresses
- Optional: roles or areas of contribution
- Ordering method (chronological or alphabetical)

### Naming Variants
- `AUTHORS.md`
- `AUTHORS`
- `CONTRIBUTORS.md`
- `CONTRIBUTORS`
- `MAINTAINERS.md`
- `CREDITS.md`
- `CREDITS`
- `OWNERS` (used for decision-makers)
- `THANKS.md`
- `MEMBERS.md`
- `CORE_TEAM.md`
- `STEWARDS.md`

### File Type Distinctions
- **AUTHORS** - Original creators and primary authors
- **CONTRIBUTORS** - All individuals who contributed
- **MAINTAINERS** - Current active maintainers
- **CORE_TEAM** - Decision-making members
- **CREDITS** - Broader acknowledgments including non-code contributions

## Best Practices

1. **Keep CHANGELOG updated** - Update with every release
2. **Be welcoming in CONTRIBUTING** - Make it easy for newcomers
3. **Credit everyone** - List all contributors, not just code authors
4. **Use consistent format** - Stick to UPPER_CASE convention
5. **Link from README** - Make these files easy to find
