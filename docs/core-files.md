# Core Files

Core files are **mandatory** for any professional repository. These files provide essential information about your project and legal framework.

## README

The README is the first file visitors see. It should **never** be plain text (.txt).

### Required Format
- Must use markup: `.md`, `.rst`, `.tex`, or similar
- Plain text `.txt` or files without extensions are unacceptable
- Markdown (`.md`) is the most widely supported format

### Required Contents
- Project description and purpose
- Quick start guide
- Installation instructions
- Basic usage information
- Links to full documentation
- License information reference
- Contribution guidelines reference (if applicable)

### Naming Variants
- `README.md`
- `README.rst`
- `readme.md`
- `ReadMe.md`

## LICENSE

**Every repository MUST have a license.** Without a license, others cannot legally use your code.

### Single License Projects
Use one of these naming patterns:
- `LICENSE`
- `LICENSE.md`
- `COPYING`
- `COPYING.md`

### Multiple Licenses
When a project uses multiple licenses, use descriptive filenames with license identifiers:
- `LICENSE-<license-name>` (e.g., `LICENSE-GPL-3.0`)
- `LICENSE-<license-name>.md` (e.g., `LICENSE-Apache-2.0.md`)

### Naming Variants
- `LICENSE` / `LICENSE.md`
- `LICENCE` / `LICENCE.md` (British spelling)
- `COPYING` / `COPYING.md`
- `LICENSE-<license-name>`

### Common License Types
- **MIT** - Permissive, simple
- **Apache-2.0** - Permissive with patent grant
- **GPL-3.0** - Copyleft, requires derivative works to be open source
- **BSD-3-Clause** - Permissive with attribution
- **CC-BY-SA-4.0** - For documentation and non-code content

## Best Practices

1. **README format matters** - Use Markdown or similar markup
2. **Always include LICENSE** - No exceptions
3. **Keep formatting consistent** - Choose UPPER_CASE convention
4. **Be explicit** - Don't assume users know your project
