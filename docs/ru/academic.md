# Файлы для академических и исследовательских проектов

Файлы, специфичные для академических, исследовательских и научных проектов.

## CITATION

Инструкции по цитированию проекта в академических работах. **Необходим для исследовательского ПО.**

### Требуемое содержание
- Предпочитаемый формат цитирования (APA, MLA, Chicago и т.д.)
- Запись BibTeX
- DOI (Digital Object Identifier), если доступен
- Версия для цитирования
- Полный список авторов с аффилиациями
- Детали публикации, если применимо
- Год выпуска
- URL проекта
- Альтернативные форматы цитирования

### Получение DOI
- Zenodo предоставляет бесплатный DOI для проектов GitHub
- Figshare — ещё один провайдер DOI
- Архивируйте релизы для постоянной возможности цитирования
- Привязывайте DOI к конкретным версиям

### Варианты названия
- `CITATION.md`
- `CITATION`
- `CITATION.cff` (Citation File Format — машиночитаемый, рекомендуется)
- `REFERENCES.md`
- `REFERENCES`

### Citation File Format (CFF)
Современный машиночитаемый стандарт, признанный GitHub и другими платформами. Содержит структурированные метаданные для автоматической генерации цитирований.

### Интеграция с платформой
GitHub распознаёт `CITATION.cff` и предоставляет виджет "Cite this repository".

## ADOPTERS

Список организаций или проектов, использующих ваше программное обеспечение. **Повышает доверие.**

### Требуемое содержание
- Названия организаций или проектов
- Опционально: логотипы компаний/организаций
- Случаи использования или области применения
- Опционально: отзывы или цитаты
- Контактная информация для представленных пользователей
- Категоризация по отрасли или случаю использования
- Масштаб развёртывания (если разрешено раскрывать)

### Когда включать
- Проекты, используемые в продакшене другими
- Проекты инфраструктуры или фреймворков
- Инструменты с известными пользователями
- Не рекомендуется для проектов на ранней стадии без пользователей

### Цель
- Демонстрирует реальное воздействие
- Повышает доверие потенциальных пользователей
- Полезно для заявок на гранты и финансирование
- Показывает зрелость проекта

### Варианты названия
- `ADOPTERS.md`
- `ADOPTERS`
- `USERS.md`
- `USERS`
- `SHOWCASE.md`
- `SHOWCASE`
- `PROJECTS.md`

## MANIFESTO

Видение и принципы проекта. **Для проектов с сильной философской позицией.**

### Требуемое содержание
- Философия проекта и основные ценности
- Принципы дизайна и обоснование
- Что проект есть и чем не является
- Долгосрочное видение и цели
- Позиции по ключевым техническим или философским дебатам
- Отличие от похожих проектов
- Руководящие принципы для принятия решений

### Когда включать
- Проекты с конкретной идеологической позицией
- Альтернативные реализации с другой философией
- Проекты, бросающие вызов устоявшимся нормам или статус-кво
- Не нужно для большинства стандартных инструментов и библиотек

### Цель
- Сообщает ценности проекта
- Привлекает единомышленников-контрибьюторов
- Проясняет направление проекта
- Устанавливает ожидания для сообщества

### Варианты названия
- `MANIFESTO.md`
- `MANIFESTO`
- `VISION.md`
- `VISION`
- `PHILOSOPHY.md`
- `PHILOSOPHY`
- `PRINCIPLES.md`

## Лучшие практики для академических проектов

1. **Всегда включайте CITATION** — Упростите правильное цитирование вашей работы
2. **Получите DOI** — Используйте Zenodo для постоянного архивирования и возможности цитирования
3. **Используйте Citation File Format** — GitHub/GitLab распознают `CITATION.cff`
4. **Обновляйте ADOPTERS** — Показывает реальное воздействие (ценно для грантов)
5. **Ссылайтесь на статьи** — Упоминайте любые публикации о проекте
6. **Версионируйте свои релизы** — Используйте семантическое версионирование для воспроизводимости
7. **Архивируйте основные версии** — Обеспечивает воспроизводимость исследований
8. **Документируйте методологию** — Объясняйте алгоритмы и подходы
9. **Предоставляйте бенчмарки** — Включайте метрики производительности и сравнения
10. **Ссылайтесь на датасеты** — Упоминайте или описывайте используемые датасеты
