# Файлы управления и сообщества

Файлы, которые устанавливают стандарты сообщества и структуры поддержки. Особенно важны для крупных или растущих проектов.

## CODE_OF_CONDUCT

Определяет ожидаемое поведение для членов сообщества. **Крайне рекомендуется** для проектов с несколькими контрибьюторами.

### Требуемое содержание
- Ожидаемые стандарты поведения
- Определения неприемлемого поведения
- Процедуры сообщения о нарушениях
- Обязанности по обеспечению соблюдения
- Последствия нарушений
- Область применения

### Стандартная рекомендация
Contributor Covenant — это отраслевой стандарт шаблона кодекса поведения, широко признанный и принятый крупными проектами.

### Почему это важно
- Устанавливает чёткие ожидания по поведению
- Предоставляет руководства по обеспечению соблюдения
- Делает сообщество приветливым и безопасным
- Часто требуется хостинг-платформами
- Предотвращает конфликты до их эскалации

### Когда включать
- Крайне рекомендуется для крупных проектов или организаций
- Рекомендуется для проектов, ищущих контрибьюторов
- Рекомендуется для любого проекта с сообществом
- Рекомендуется даже для небольших проектов

### Варианты названия
- `CODE_OF_CONDUCT.md`
- `CODE_OF_CONDUCT`
- `CONDUCT.md`
- `CONDUCT`
- `CoC.md`

## SUPPORT

Объясняет, как пользователи могут получить помощь с проектом.

### Требуемое содержание
- Где задавать вопросы (форумы, чат-платформы, списки рассылки)
- Как сообщать об ошибках
- Ожидаемое время ответа
- Варианты платной поддержки (если доступны)
- Ресурсы самопомощи и ссылки на документацию
- Каналы сообщества
- Контактная информация для мейнтейнеров

### Варианты названия
- `SUPPORT.md`
- `SUPPORT`
- `HELP.md`
- `HELP`
- `CONTACT.md`
- `CONTACT`
- `TROUBLESHOOTING.md`
- `TROUBLESHOOTING`

### Ключевая информация для предоставления
- Различие между ошибками и вопросами поддержки
- Доступные каналы поддержки
- Ожидания по времени ответа
- Руководства сообщества для запросов поддержки

## GOVERNANCE

Описывает, как принимаются решения в проекте. **Критически важно для крупных проектов и организаций.**

### Требуемое содержание
- Процесс и процедуры принятия решений
- Определения ролей и обязанностей
- Процедуры голосования (если применимо)
- Процесс становления мейнтейнером или членом основной команды
- Процедуры разрешения конфликтов
- Процесс внесения поправок в само управление
- Расписание и процедуры встреч (если применимо)
- Структура технического руководящего комитета (если применимо)

### Когда включать
- Крайне рекомендуется для организаций (не только отдельных проектов)
- Необходимо для очень крупных проектов
- Рекомендуется для проектов с несколькими мейнтейнерами
- Не нужно для небольших личных проектов

### Варианты названия
- `GOVERNANCE.md`
- `GOVERNANCE`
- `CHARTER.md`
- `CHARTER`
- `DECISIONS.md`
- `DECISIONS`

### Ключевые элементы
- Чёткая иерархия и структура власти
- Прозрачные процессы принятия решений
- Определённые пути для участия сообщества
- Планирование преемственности для руководящих ролей

## Лучшие практики

1. **Начинайте с CODE_OF_CONDUCT рано** — Не ждите, пока возникнут проблемы
2. **Используйте Contributor Covenant** — Он хорошо протестирован и широко признан
3. **Сделайте SUPPORT доступным** — Ссылайтесь заметно из README
4. **GOVERNANCE предотвращает конфликты** — Определите процессы до возникновения споров
5. **Будьте реалистичны** — Не создавайте управленческую нагрузку, которую не можете поддерживать
6. **Регулярно обновляйте** — Пересматривайте и обновляйте по мере роста проекта
7. **Будьте прозрачными** — Сделайте все документы по управлению легко обнаруживаемыми
