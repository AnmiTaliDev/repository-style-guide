# Файлы для контрибьюторов и истории проекта

Эти файлы помогают контрибьюторам понять, как участвовать в вашем проекте, и отслеживать его эволюцию во времени.

## CONTRIBUTING

Руководство по внесению вклада в проект. Необходимо для проектов, принимающих внешние контрибуции.

### Требуемое содержание
- Процесс отправки issues и запросов на новые функции
- Процесс и требования к pull request'ам
- Руководства и соглашения по стилю кода
- Требования и процедуры тестирования
- Инструкции по настройке окружения для разработки
- Соглашения по именованию веток
- Руководства по сообщениям коммитов

### Варианты названия
- `CONTRIBUTING.md`
- `CONTRIBUTING`
- `HACKING.md` (распространено в проектах GNU/FSF)
- `HACKING`
- `DEVELOPMENT.md`
- `DEVELOPMENT`

### Когда включать
- Проекты, принимающие контрибуции
- Проекты с открытым исходным кодом
- Командные проекты с несколькими разработчиками
- Опционально для личных/экспериментальных проектов

## CHANGELOG

Запись всех значительных изменений в проекте. Критически важен для пользователей и мейнтейнеров для отслеживания эволюции проекта.

### Требуемое содержание
- Номера версий с датами релизов
- Изменения, организованные по категориям (Added, Changed, Deprecated, Removed, Fixed, Security)
- Значительные исправления ошибок
- Выделенные несовместимые изменения
- Предупреждения об устаревании

### Стандарты формата
- Используйте формат Keep a Changelog
- Организуйте хронологически (сначала новые)
- Включайте номера версий, следующие семантическому версионированию
- Формат даты: ISO 8601 (YYYY-MM-DD)

### Варианты названия
- `CHANGELOG.md`
- `CHANGELOG`
- `HISTORY.md`
- `HISTORY`
- `NEWS.md`
- `NEWS`
- `RELEASES.md`
- `RELEASES`
- `CHANGES.md`
- `CHANGES`

## AUTHORS / CONTRIBUTORS / MAINTAINERS

Список людей, внёсших вклад в проект.

### Требуемое содержание
- Имена контрибьюторов
- Опционально: адреса электронной почты
- Опционально: роли или области вклада
- Метод упорядочения (хронологический или алфавитный)

### Варианты названия
- `AUTHORS.md`
- `AUTHORS`
- `CONTRIBUTORS.md`
- `CONTRIBUTORS`
- `MAINTAINERS.md`
- `CREDITS.md`
- `CREDITS`
- `OWNERS` (используется для лиц, принимающих решения)
- `THANKS.md`
- `MEMBERS.md`
- `CORE_TEAM.md`
- `STEWARDS.md`

### Различия типов файлов
- **AUTHORS** — Оригинальные создатели и основные авторы
- **CONTRIBUTORS** — Все лица, внёсшие вклад
- **MAINTAINERS** — Текущие активные мейнтейнеры
- **CORE_TEAM** — Члены, принимающие решения
- **CREDITS** — Более широкие благодарности, включая вклад, не связанный с кодом

## Лучшие практики

1. **Обновляйте CHANGELOG** — Обновляйте с каждым релизом
2. **Будьте приветливы в CONTRIBUTING** — Сделайте это простым для новичков
3. **Отдавайте должное всем** — Перечисляйте всех контрибьюторов, не только авторов кода
4. **Используйте единообразный формат** — Придерживайтесь конвенции UPPER_CASE
5. **Ссылайтесь из README** — Сделайте эти файлы легко находимыми
