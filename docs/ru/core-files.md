# Основные файлы

Основные файлы являются **обязательными** для любого профессионального репозитория. Эти файлы предоставляют важную информацию о вашем проекте и правовую основу.

## README

README — это первый файл, который видят посетители. Он **никогда** не должен быть простым текстом (.txt).

### Требуемый формат
- Должен использовать разметку: `.md`, `.rst`, `.tex` или подобные
- Простой текст `.txt` или файлы без расширений неприемлемы
- Markdown (`.md`) — наиболее широко поддерживаемый формат

### Требуемое содержание
- Описание и цель проекта
- Руководство по быстрому старту
- Инструкции по установке
- Основная информация об использовании
- Ссылки на полную документацию
- Ссылка на информацию о лицензии
- Ссылка на руководство по внесению вклада (если применимо)

### Варианты названия
- `README.md`
- `README.rst`
- `readme.md`
- `ReadMe.md`

## LICENSE

**Каждый репозиторий ДОЛЖЕН иметь лицензию.** Без лицензии другие не могут легально использовать ваш код.

### Проекты с одной лицензией
Используйте один из этих шаблонов именования:
- `LICENSE`
- `LICENSE.md`
- `COPYING`
- `COPYING.md`

### Несколько лицензий
Когда проект использует несколько лицензий, используйте описательные имена файлов с идентификаторами лицензий:
- `LICENSE-<название-лицензии>` (например, `LICENSE-GPL-3.0`)
- `LICENSE-<название-лицензии>.md` (например, `LICENSE-Apache-2.0.md`)

### Варианты названия
- `LICENSE` / `LICENSE.md`
- `LICENCE` / `LICENCE.md` (британское написание)
- `COPYING` / `COPYING.md`
- `LICENSE-<название-лицензии>`

### Распространённые типы лицензий
- **MIT** — Разрешительная, простая
- **Apache-2.0** — Разрешительная с патентным грантом
- **GPL-3.0** — Копилефт, требует, чтобы производные работы были открытым исходным кодом
- **BSD-3-Clause** — Разрешительная с указанием авторства
- **CC-BY-SA-4.0** — Для документации и некодового контента

## Лучшие практики

1. **Формат README имеет значение** — Используйте Markdown или подобную разметку
2. **Всегда включайте LICENSE** — Без исключений
3. **Сохраняйте единообразие форматирования** — Выберите конвенцию UPPER_CASE
4. **Будьте явными** — Не предполагайте, что пользователи знают ваш проект
